    Introduction

    A Rails API that provides a bucket list service. Its really easy. Following the next few steps should get you up and running in no time.

    Sign up

    Visit Sign up Page to sign up and register your data as a user on the app. Once you have done this, you are now ready to make requests.

    Authentication

    Unlike conventional web apps, APIs are authenticated differently. Logging in will return an authentication token which you would use to make further calls to the api. The token expires after 2 hours so you would have to log in again.

    To get your authentication token, you need to structure your request like this:

    curl -X POST -d email='your@email.com'&password='password'
  http://mybucketapi.herokuapp.com/auth/login

    You’ll get a json response like this:

    {
  "auth_token": "ashbdjfaskjfdl.ksldhfkjsadhbjahiudhiasf.suduskbajfbksjdbfksd"
}

    MyBucketApi uses this auth_token to allow access to the API. MyBucketApi expects for the auth_token to be included in all API requests to the server in a header that looks like the following:

    Authorization: ashbdjfaskjfdl.ksldhfkjsadhbjahsf.suduskbajfbksjdbfksd

    You must replace ashbdjfaskjfdl.ksldhfkjsadhbjahiudhiasf.suduskbajfbksjdbfksd with your personal auth token.

    Available End Points

    END POINT FUNCTIONALITY PUBLIC ACCESS
    POST /auth/login Logs a user in true
    GET /auth/logout Logs out a user false
    POST /bucketlists Create a new bucketlist false
    GET /bucketlists Retrieve all bucketlists false
    GET /bucketlists/:id Retrieve a particular bucketlist false
    PUT /bucketlists/:id Update a bucketlist false
    DELETE /bucketlists/:id Delete a bucketlist false
    POST /bucketlists/:id/items Create a new item in a bucketlist false
    GET /bucketlists/:id/items Retrieve all the items in a bucketlist false
    GET /bucketlists/:id/items/:id Retrieve a particular item in a bucketlist false
    PUT /bucketlists/:id/items/:id Update an item in a bucketlist false
    DELETE /bucketlists/:id/items/:id Delete an item in a bucketlist false

    Bucketlists

    Create a Bucketlist

    curl -X POST
"http://mybucketlistapi.herokuapp.com/bucketlists/"
  -H "Authorization: ashbdjfaskjfdl.ksldhfkjsadhbjaiasf.suduskbajfbksjdbfksd"
  -d name=Become+World+Class

    The above command returns JSON structured like this:

    {
  "id": 23,
  "name": "'Become World Class",
  "date_created": "2016-10-28T16:17:43.046Z",
  "date_updated": "2016-10-28T16:17:43.046Z",
  "created_by": 1,
  "items": []
}

    This endpoint create a new bucketlist and returns the same bucketlist.

    HTTP Request

    POST http://mybucketlistapi.herokuapp.com/bucketlists/name=some+name

    URL Parameters

    Parameter Default Description
    name nil The new name you want to give the bucketlist

    Query Parameters

    Parameter Default Description
    page 0 Should be given a positive integer value. Works together with the limit parameter
    limit 20 Should also be given a positive integer value
    q nil should be a string that would match a collection of bucketlists
    A request like this http://mybucketlistapi.herokuapp.com/bucketlists?page=2&limit=10 will return the 11th to 20th bucketlist elements.

    Errors

    This error section is stored in a separate file in `includes/_errors.md`. Slate allows you to optionally separate out your docs into many files…just save them to the `includes` folder and add them to the top of your `index.md`’s frontmatter. Files are included in the order listed.

    Bucketlist API uses the following error codes:

    Error Code Meaning
    400 Bad Request – Your request isn't properly structured
    401 Unauthorized – Your authentication key is wrong
    403 Forbidden – The bucketlist requested is not open for public access
    404 Not Found – The specified bucketlist could not be found
